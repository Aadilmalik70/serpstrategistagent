Focus on current-site opportunities, not stale historical URLs.

Content planning rules:

1. Prefer refresh opportunities when an existing live page is already relevant.
2. Only suggest net-new content when there is a real gap.
3. Exclude legacy URLs unless they are confirmed live and strategic.
4. Do not auto-publish in v1.

Return:

1. refresh candidates
2. new-content candidates
3. supporting queries
4. why each is current-site relevant

Call out when the data is too sparse to justify net-new content.
