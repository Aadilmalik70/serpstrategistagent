Return indexability, likely issue class, supporting evidence, and recommended next action.

Inspection flow:

1. Confirm canonical target and current URL form.
2. Check whether Search Console evidence is current or historical.
3. Check ranking or SERP context if the URL appears externally.
4. Inspect the local repo or WordPress surface if available.
5. Decide whether the issue is technical, content, analytics, or false-positive historical noise.

If the URL is likely legacy, say that before recommending a fix.
